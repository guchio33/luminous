# ✦ Luminous — 要件定義書

> 暗闇の中で光り輝く夢とやりたいことを共有するアプリ

---

## 1. コンセプト

### アプリ名

**Luminous（ルミナス）**

### ビジョン

「少しでも人生が楽しいな、幸せだなと思うきっかけを作りたい」

夢ややりたいことを言葉にしてシェアすることで、同じ光を探す誰かと静かに繋がれる場所。  
アプリは「幸せの答え」を与えない。自分の中にある光に**気づくきっかけ**を作る。

### キャッチコピー

- **JP**: 夢を言葉にすると、世界が変わる
- **EN**: Your dreams are stars in someone else's darkness

---

## 2. ターゲットユーザー

- 幸せとは何かと悩んでいる人
- 同じ毎日を繰り返していて、何か変わりたいと思っている人
- 夢がないわけじゃない、ただ忘れているだけの人
- 一人では動けないが、誰かの「私も」という言葉で動ける人

### ユーザーの心の状態

| 状態             | Luminous が提供するもの    |
| ---------------- | -------------------------- |
| 幸せがわからない | 小さな光を見つける習慣     |
| 毎日が同じ       | やりたいことで日常に彩りを |
| 変わりたい       | 夢を言葉にする最初の一歩   |
| 孤独な気持ち     | 「私も」という共鳴と繋がり |

---

## 3. 主要機能

### 3-1. ドリームポスト

夢ややりたいことを短い言葉で投稿する。  
大きさは関係なし。「南米を旅したい」も「抹茶パフェを食べに行く」も同じ価値を持つ。

### 3-2. 共鳴ボタン「✦ 私も！」

いいねではなく「私もそれ夢だった！」と共鳴できるアクション。  
孤独な夢が、誰かとつながる瞬間になる。

### 3-3. やりたいことリスト

夢より小さな日常のワクワクを記録。  
達成したら光るエフェクトで、小さな達成感を演出する。

### 3-4. 今日の小さな光

一日の終わりに「今日ちょっと良かったこと」を一言投稿。  
ネガティブな日でも、小さな光を探す習慣をつける。

## 6. DB 設計

### `auth.users`（Supabase 自動生成）

Supabase が認証時に自動で管理するテーブル。

### `profiles`

| カラム     | 型        | 説明                      |
| ---------- | --------- | ------------------------- |
| id         | uuid      | auth.users と紐づく（PK） |
| username   | text      | ユーザー名                |
| avatar_url | text      | アバター画像 URL          |
| created_at | timestamp | 作成日時                  |

### `dreams`

| カラム     | 型        | 説明                 |
| ---------- | --------- | -------------------- |
| id         | uuid      | PK                   |
| user_id    | uuid      | profiles.id と紐づく |
| content    | text      | 夢の内容             |
| created_at | timestamp | 投稿日時             |

### `resonances`（「私も！」）

| カラム     | 型        | 説明               |
| ---------- | --------- | ------------------ |
| id         | uuid      | PK                 |
| dream_id   | uuid      | dreams.id と紐づく |
| user_id    | uuid      | 共鳴したユーザー   |
| created_at | timestamp | 共鳴日時           |

> ユーザー登録時に`auth.users`のトリガーで`profiles`に自動レコード作成。

## 7. 世界観・デザイン方針

- **カラー**: ダークネイビー × ゴールド × ホワイト
- **フォント**: Cormorant Garamond（見出し）+ Noto Serif JP（本文）
- **モチーフ**: 夜空・星・光
- **UI**: 投稿が星のように夜空に浮かぶイメージ。夢が増えるほど自分の空が星でいっぱいになる。
- **トーン**: 静かで、温かく、押しつけがましくない

## 8. 開発ロードマップ（ミニマム）

```
Step 1  環境構築
        create-next-app → Supabase プロジェクト作成 → Vercel 連携

Step 2  認証
        Supabase Auth でメール or Google ログイン実装

Step 3  DB構築
        profiles / dreams / resonances テーブル作成・RLS設定

Step 4  画面実装
        TOP → タイムライン → 投稿ページ → プロフィール

Step 5  デプロイ
        GitHub push → Vercel 自動ビルド・公開
```
